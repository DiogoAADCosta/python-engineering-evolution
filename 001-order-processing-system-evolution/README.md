This evolution path started as a simple function exercise
focused on calculating order totals.

As new requirements were introduced,
the system progressively evolved into a more structured
order processing pipeline.

Throughout the exercises, new challenges emerged:
- chained processing
- conditional business rules
- batch processing
- responsibility separation
- reusable workflows
- reporting systems

The goal of this path was not only learning functions,
but understanding how growing complexity naturally creates
the need for better structure and software design.
